# **1. Understanding the UI Structure**

Before writing any prompt or code, the most important step is breaking down the UI into logical components. The showcased real estate app interface contains several distinct sections, each with its own layout considerations.

---

## **1.1 Header Section**
The top of the screen features a **custom header** built using a `Row` composable.  
It includes:

- A user profile image on the left  
- A notification bell icon on the right  

This layout establishes a clean, modern top bar without relying on a default `TopAppBar`.

---

## **1.2 Search Bar**
Directly below the header is a **custom search bar**, not a standard text field.

Key characteristics:

- Rounded corners with a **30dp radius**
- Soft, modern visual style
- Designed to blend seamlessly with the rest of the UI

This component is typically built using a `Surface` or `Box` with custom shapes and padding.

---

## **1.3 Category Section**
Next is a horizontally scrollable **category selector**, implemented using a `LazyRow`.

Each category—such as:

- House  
- Apartment  
- Villa  

—is treated as its own composable for modularity and reusability.

This section allows users to filter properties visually and interactively.

---

## **1.4 Property Card (Main Component)**
The property card is the centerpiece of the UI and the most complex element.

It includes:

### **Image Area**
- Wrapped in a `Box` to allow stacking
- A floating price tag positioned on top of the image using alignment modifiers
- Rounded corners applied via `clip(RoundedCornerShape)`

### **Property Details**
Below the image:

- Title (e.g., “Modern Family House”)
- Address
- A row of meta chips showing:
  - Number of bedrooms
  - Number of bathrooms
  - Square footage

### **Scrolling Behavior**
All cards are placed inside a `LazyColumn` to support smooth vertical scrolling.

Padding is added at the bottom (e.g., 100dp) to prevent the last item from being hidden behind the bottom navigation bar.

---

## **1.5 Floating Bottom Navigation Bar**
Unlike a standard edge‑to‑edge navigation bar, this design uses a **floating bottom navigation**:

- Detached from screen edges  
- Dark background  
- Rounded corners  
- Elevated, premium appearance  

This enhances the modern aesthetic and improves usability.
